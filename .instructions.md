Use Unreal Engine C++ conventions for project source, configuration, and content.

- Apply these guidelines to project files under `Source/`, `Config/`, `Content/`, and project-level build/source configuration.
- Follow the repository's `.clang-format` style for C++:
  - 4-space indentation, no tabs
  - Allman brace style
  - `SpaceBeforeParens: ControlStatements`
  - `PointerAlignment: Left`
- Prefer Unreal Engine idioms when editing C++ code: `UCLASS`, `UPROPERTY`, `UFUNCTION`, `TWeakObjectPtr`, `TSubclassOf`, `FName`, `FText`, and engine-safe object lifetime patterns.
- Prefer TObjectPtr in member declarations and raw pointer for local variables and arguments when appropriate.
- Use Unreal Engine's logging system (`UE_LOG`) for debug output.
- When creating or editing class, expose class interface to Blueprints when it makes sense for the project. Use categories that match the class name for better organization in the editor.
- Do not modify engine files under `UE5/Engine/` unless explicitly requested.
- Keep generated or auto-generated code untouched; only edit project-specific source.
- When answering user requests, be concise, technical, and include only the minimal relevant code snippets.
- If the scope of a rule is unclear, ask for clarification before applying broad changes.
- Follow best coding practices for C++ and Unreal Engine, such as using `const` where appropriate, avoiding raw pointers when possible, and adhering to the engine's memory management conventions.
- When working on architectural or design tasks, try to find best pattern that fits the problem, such as using components for modular behavior, using interfaces for decoupling, or using design patterns like Factory or Observer.
